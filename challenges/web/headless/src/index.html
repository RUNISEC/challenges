API Reference

Explore our endpoints and start building.

GET /api/v1/status

Returns the current status of the API and server health information.

GET /api/v1/users

Retrieve a list of users. Requires authentication token.

POST /api/v1/auth/login

Authenticate with the API and receive a bearer token.

# Example: Check API status
curl http://localhost:3000/api/v1/status

# Tip: Always check the full response,
# not just the body. Headers can contain
# useful metadata about the API.

Pro tip: Our API returns useful debugging information in the response headers. Make sure to inspect them when troubleshooting.
